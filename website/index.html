2. Configure

3. Run

./target/release/mobium-server

4. Connect your client to ws://your-ip:8443

FAQ

How is this different from mainstream chat platforms?
Mainstream platforms can read every message you send. Mobium encrypts everything client-side before it leaves your device. The server is mathematically unable to decrypt your content.
How is this different from other encrypted messengers?
Mobium uses the same proven cryptographic primitives but adds community features: persistent channels, voice chat rooms, screen sharing, and server-based message history. And you can self-host.
What if I lose my device?
When you create your identity, you get a 24-word BIP39 recovery phrase. Write it down. That's the only way to recover your keys ‚Äî the server can't help you.
Can the server operator read my messages?
No. Not even with full database access. Messages are encrypted with keys derived from X3DH and Double Ratchet. The server only sees opaque ciphertext blobs.
Does file sharing go through the server?
No. Files transfer peer-to-peer via WebRTC data channels with AES-256-GCM encryption. The server only relays the signaling handshake.
What happens when someone leaves a channel?
All remaining members rotate their sender keys. The departed member's keys are destroyed. They cannot decrypt any future messages (forward secrecy).
