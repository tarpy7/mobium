Everything encrypted, nothing exposed

Every message, call, and file transfer is encrypted before it leaves your device.

💬

Encrypted Messages

Messages are encrypted on your device and relayed as opaque blobs through the server. Offline recipients get them on reconnect. Double Ratchet gives every message a unique key.

👥

Groups & Channels

Ad-hoc group DMs and persistent channels. Sender Keys with bucket padding so the server can't see message sizes. Keys rotate when members leave.

🎙️

Voice Calls

≤4 people: direct P2P mesh — audio never touches the server. 5–40: AES-256-GCM encrypted relay through the server. Capped at 40 per channel.

🖥️

Screen Sharing

P2P in DMs (server never sees it), encrypted relay in channels. Quality presets from 360p to 1080p.

📁

File Transfer

DM file transfers go directly between peers via WebRTC — the server never sees file data. AES-256-GCM encrypted chunks with a client-side NSFW filter.

🧅

Tor Integration

Built-in Arti client hides your IP from the server. Run the server as a .onion hidden service for full anonymity on both ends.

🔑

No Accounts

Ed25519 keypair is your identity. No email, no phone number. 24-word BIP39 mnemonic recovery. Age verified locally — nothing stored.

🏠

Self-Hosted

Single binary on a Raspberry Pi or VPS. ~9MB RAM. SQLite. No Docker needed. Your infrastructure, your rules.

🛡️

Content Safety

Client-side NSFW filter runs entirely on your device. If the model fails to load, all image/video transfers are blocked. Nothing leaves your device.

Zero-knowledge by design

The server is a dumb pipe. It stores encrypted blobs and routes them. That's it.

1. Clone and build

2. Configure

cp .env.example .env && mkdir -p data

3. Run

./target/release/mobium-server

4. Connect your client to ws://your-ip:8443

FAQ

How is this different from mainstream chat platforms?
Mainstream platforms can read every message you send. Mobium encrypts everything client-side before it leaves your device. The server is mathematically unable to decrypt your content.
How is this different from other encrypted messengers?
Mobium adds community features — persistent channels, group calls, screen sharing — while keeping everything E2E encrypted. Plus Tor integration and self-hosting.
What if I lose my device?
Your 24-word BIP39 recovery phrase is the only way to restore your keys. Write it down. The server can't help you recover — by design.
Can the server operator read my messages?
No. Not even with full database access. Messages are encrypted with keys derived from X3DH and Double Ratchet. The server only sees opaque ciphertext.
Does file sharing go through the server?
In DMs, no — files transfer directly between peers via WebRTC with AES-256-GCM encryption. The server never sees file data. Channel/group file sharing is not yet supported. A client-side content filter scans media locally before sending.
Is there an age requirement?
Users must be 13 or older. Age verification happens entirely on your device — nothing is stored or sent to the server.
